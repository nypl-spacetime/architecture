NYC Space/Time Directory - Architecture

This diagram shows the current state of the evolving technical architecture of the NYC Space/Time Directory project. The diagram shows how tools and components work together, and how they share data. It does not show the final state of the project â€” new blocks will be added as the project procedes and work is started on new datasets, data processing tools and user interfaces. Parts of the Space/Time Directory are based on Histograph, a search engine for historical place names built by Waag Society.

For more information about the NYC Space/Time Directory, see spacetime.nypl.org.

In this diagram, only code and data components of the Space/Time Directory project are shown. Other, and equally important parts of the project (e.g. outreach, meetups, hackathons, collaborations), are not shown in this diagram.

Finished components are green, components on which work has not started are white, and components on which we are currently working are shaded green according to their finished state. External, non-Space/Time components are white with a dashed line. For components with associated GitHub repositories, the number of open issues are shown in the red badges.

This architecture diagram itself can be found on GitHub too. Please open an issue if the diagram is incorrect or out of date. And it's easy to make a similar diagram for your own project!
